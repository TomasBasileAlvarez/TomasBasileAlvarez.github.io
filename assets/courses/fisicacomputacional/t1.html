4 Algoritmo de Fracciones continuas¶

Para la soluci√≥n de ecuaciones cuadr√°ticas, en ocasiones se tienen fuertes singularidades y la f√≥rmula ya no es funcional. Una manera de darle la vuelta a estas singularidades es reescribir la ecuaci√≥n a evaluar en t√©rminos de una expansi√≥n Taylor. Otra posibilidad es utilizar las llamadas fracciones continuas, que pueden verse como generalizaciones de una expansi√≥n de Taylor. Un enfoque posible de las fracciones continuas es la sustituci√≥n sucesiva. Vamos a ilustrar esto con un ejemplo simple:

$x^2 + 4x-1=0$

que puede reescribirse como:

$x = \dfrac{1}{4+x}$

que, a su vez, podr√≠a representarse a trav√©s de un proceso iterativo de sustituci√≥n:

$x_{n+1} = \dfrac{1}{4+x_n}$

con $x_0 = 0$. Esto significa que si tenemos: $x_1 = \dfrac{1}{4}$ $x_2 = \dfrac{1}{4+ \dfrac{1}{4}}$ $x_3 = \dfrac{1}{4+ \dfrac{1}{\dfrac{1}{4}}}$

Esto puede ser reescrito de forma compacta como:

1. Escribe un programa que resuelve la ecuaci√≥n (1) con la implementaci√≥n del algoritmo de fracci√≥n continua y resuelva iterativamente. La soluci√≥n exacta es x = 0.23607 ¬øCu√°ntas iteraciones necesitamos para tener 3 cifras decimales correctas?

# Vamos a definir una funci√≥n que toma x_n y nos da x_{n+1} 
# Para ello usamos la funci√≥n iterativa que nos da el enunciado

#Toma como entrada un n√∫mero x
def siguiente(x):
    # Le aplica la funci√≥n que vimos 1/(4+x)
    y = 1/(4+x)
    # Regresa el resultado
    return(y)

# Definimos la funci√≥n iterar, que va a realizar toda la iteraci√≥n hasta
# Llegar a la precisi√≥n deseada

# La funci√≥n toma dos entradas, el valor de x inicial y la precisi√≥n que queremos
def iterar(x,e=1e-3):
    #inicializamos el contador
    contador = 0
    # Calculamos el primer siguiente elemento de la sucesi√≥n
    xn = siguiente(x)
    # Y aumentamos el contador porque ya hicimos un paso
    contador += 1
    
    # Ahora, mientras la diferencia entre un valor y el siguiente
    # Sea mayor al error deseado
    while abs(xn-x)>e:
        # El x siguiente pasa a ser el x actual
        x = xn
        
        # Y calculamos el siguiente elemento de la sucesi√≥n despu√©s de xn
        xn = siguiente(xn)
        # Vamos aumentando el contador
        contador+=1
    # Regresamos el √∫ltimo valor de la secuencia y el contador
    return([xn,contador])
    
# Calculamos el resultado de empezar desde 0 la iteraci√≥n
# Con una precisi√≥n de 0.001
res = iterar(0,1e-3)

print('Resultado final:', res[0])
print('Cantidad de pasos:', res[1])

2.Generaliza el c√≥digo para que el usuario introduzca los coeficientes de la funci√≥n y el n√∫mero de precisi√≥n que desea obtener.

Supongamos que la ecuaci√≥n de segundo grado es de la forma: $ax^2 + bx + c = 0$

Entonces, se tiene que $ax^2+bx = -c$
Por lo que $x(ax+b) = -c$ y entonces $x = \dfrac{-c}{ax+b}$

Por lo que la f√≥rmula recursiva es: $x_{n+1} = \dfrac{-c}{ax_n+b}$

Ahora implementamos este m√©todo de una forma muy similar a la parte anterior

def iterar(x,a,b,c,e=1e-6):
    # Si x inicial es -b/a, la funci√≥n siguiente va a tener que dividir entre 0
    # por lo que cambiamos levemente el x
    if x == -b/a:
        x+=0.1
        
    # inicializamos el contador
    contador = 0
    # Calculamos el primer siguiente de la lista
    xn = siguiente(x,a,b,c)
    contador+=1

    # Agregamos tambi√©n un l√≠mite al contador (de 2000 en este caso)

    while abs(xn-x)>e and contador < 2000:

        x = xn
        # calculamos el siguiente elemento y aumentamos el contador
        xn = siguiente(xn,a,b,c)
        contador+=1
    
    # Al terminar el while, si la secuencia logr√≥ converger (no se pas√≥ del l√≠mite)
    if contador <2000:

    # Si la secuencia no converge, imprimimos un mensaje de error
    else:

Si la funci√≥n es $ax^2 + bx + c =0$, distinguimos varios casos:

  1. Si b = 0, la secuencia no converge
  2. Si $b \neq 0$ y el discriminante $\sqrt{b^2-4ac}$ es negativo, la secuencia diverge (pues no puede converger a una ra√≠z ya que ambas son imaginarias)
  3. Si $b \neq 0$ y el discriminante es 0, la secuencia converge a la √∫nica ra√≠z de multiplicidad 2
  4. Si $b \neq 0$ y el discriminante es positivo, la secuencia converge a alguna de ellas

from numpy import *

# Definimos la funci√≥n
def transpuesta(A):
    # contamos el n√∫mero de renglones y columnas de la matriz
    ren_A = len(A)
    col_A = len(A[0,:])
    
    # Inicializamos una nueva matriz con las dimensiones al rev√©s
    # y del mismo tipo que A
    AT = zeros([col_A,ren_A],A.dtype)
    
    # Iteramos sobre las posiciones de nuestra nueva matriz AT
    for i in range(col_A):
        for j in range(ren_A):
            # El valor de AT en [i,j] es igual al de A en [j,i]
            AT[i,j] = A[j,i]
    # Regresamos el resultado
    return(AT)

def suma(A,B):
    # Calculamos las dimensiones de las matrices
    ren_A = len(A)
    col_A = len(A[0,:])
    ren_B = len(B)
    col_B = len(B[0,:])
    
    
    # Si las dimensiones de las matrices no coinciden
    # no se puede realizar la operaci√≥n, regresamos un mensaje de error
    if ren_A != ren_B or col_A != col_B:
        return('No es posible hacer la suma, las dimensiones tienen que ser iguales')
    
    # Creamos una matriz con las dimensiones de A
    # El tipo de esta matriz va a ser el promote_types del tipo de A y B
    # La funci√≥n promote_types de numpy regresa un tipo de datos que pueda
    # contener a los tipos A.dtype y B.dtype sin alterar sus valores
    res = zeros([ren_A,col_A],promote_types(A.dtype,B.dtype))
    
    # Vamos iterando sobre las posiciones de la matriz
    for i in range(ren_A):
        for j in range(col_A):
            # El elemento de la matriz res es la suma de los de A y B
            res[i,j]=A[i,j]+B[i,j]
    return(res)

# Definimos la funci√≥n mult que toma una matriz A y un n√∫mero c
def mult(A,c):
    # Calculamos las dimensiones
    ren_A = len(A)
    col_A = len(A[0,:])
    # Generamos una nueva matriz de la misma dimensi√≥n que A
    # Con el tipo que pueda contener al tipo de A y el de c
    res = zeros([ren_A,col_A],promote_types(A.dtype,type(c)))
    
    # Vamos iterando sobre las posiciones
    for i in range(ren_A):
        for j in range(col_A):
            # En la posici√≥n i,j ponemos el resultado de c por 
            # la posici√≥n i,j de A
            res[i,j] = c*A[i,j]
    return(res)
